# Tostador Café: Proyecto Grupal de Análisis de Datos y SQL
## Descripción del Proyecto
Este repositorio contiene un ejercicio grupal centrado en el ciclo completo de gestión de datos para una empresa tostadora de café. El proyecto abarca desde la recepción de datos en bruto hasta la creación de un sistema de información estructurado y optimizado para el análisis de negocio.

## 🎯 Objetivos del Proyecto

- **Modelado de Datos:** Construcción de un modelo relacional robusto que refleje la operativa de la empresa.
- **Calidad del Dato:** Detección y análisis de problemas, inconsistencias y errores en los datos iniciales (dataset "raw").
- **Normalización:** Aplicación de técnicas de normalización hasta la Tercera Forma Normal (3FN) para eliminar redundancias y asegurar la integridad.
- **Análisis SQL:** Desarrollo de consultas complejas para responder preguntas estratégicas de negocio.

## 🛠️ Tecnologías y Herramientas

- **SQL:** Lenguaje fundamental utilizado para la creación de tablas, normalización y consultas analíticas.
- **Jupyter Notebook:** Entorno principal de desarrollo y documentación, que constituye el 98.2% del código del proyecto.
- **HTML:** Utilizado para la presentación de resultados y visualización de dashboards.

## 📂 Estructura del Repositorio
El proyecto se organiza mediante los siguientes archivos clave:

    tostador-cafe-raw.db          : Base de datos original con la información sin procesar para el análisis inicial.
    CHULETA-RAW-A-NORMALIZADO.md  : Guía metodológica que documenta el proceso de transformación de los datos.
    Imagen_esquema_tablas.png     : Representación visual del diagrama entidad-relación del proyecto.
    creacion tablas.sql           : Script DDL con la estructura de la base de datos normalizada.
    preguntas Tostador.sql        : Repositorio de consultas SQL diseñadas para extraer métricas de negocio.
    Prueba presentacion.ipynb     : Notebook principal que integra el código, las visualizaciones y la explicación del flujo de trabajo.

## 💡 Habilidades Demostradas

- **Arquitectura de Datos:** Capacidad para transformar datos desestructurados en modelos relacionales eficientes bajo estándares de normalización (3FN).
- **Resolución de Problemas:** Identificación proactiva de anomalías en conjuntos de datos brutos.
- **Pensamiento Analítico:** Traducción de necesidades de negocio en consultas técnicas precisas para la toma de decisiones.
- **Documentación Técnica:** Elaboración de recursos de apoyo y guías claras para la reproducibilidad del proyecto.
